# Financas

Single-user, browser-based, investment-first personal finance manager
(Go + PostgreSQL, layered "onion", Docker → Azure). This repository currently
contains the **Story 1.1 scaffold**: the layered package structure, a running
chi server with a `/healthz` probe, the codegen tooling, and a single-image
Docker setup with local PostgreSQL.

## Prerequisites

- **Go 1.26.x** (developed against 1.26.3; the spine targets 1.26.4)
- **Docker** + **Docker Compose** (for the local app + PostgreSQL 18)
- **Node.js / npm** (only for the Tailwind CSS v4 build toolchain, pinned in
  `package.json`; `make css` installs it on first run)

The `templ` and `sqlc` generators are invoked through Go (`go tool` / `go run`),
so no separate global installs are required.

## Layout (layered "onion" — AD-1)

```
cmd/server/            # main: config, wiring, http.ListenAndServe
internal/
  domain/              # entities + ALL derivations; imports nothing project-internal
  money/               # decimal Money type, Currency, Convert()  (Story 1.2)
  service/             # use-cases; owns DB transactions (the only mutator)
  store/               # pgx + sqlc-generated queries
  http/                # chi handlers, middleware, templ render
web/                   # *.templ views + embedded static assets (CSS, JS)
db/migrations/         # goose *.sql
db/query/              # sqlc source queries
```

**Dependency rule (AD-1):** dependencies point inward only —
`http → service → store`, all → `domain`/`money`, and `domain`/`money` import
nothing project-internal. A handler never calls the store directly; the store
never imports `service`; `http` performs no financial math (AD-10).

## Common commands

All wrapped in the `Makefile` (run `make help` for the full list):

| Command | What it does |
| --- | --- |
| `make run` | Run the server locally (`PORT` defaults to 8080) |
| `make test` | Run the Go test suite (DB integration tests skip without a database) |
| `make vet` | `go vet ./...` |
| `make nofloat` | NFR-5 guard: fail if `float32`/`float64` appears in the financial core |
| `make generate` | Run codegen: `templ generate` + `sqlc generate` |
| `make css` | Build `web/static/css/app.css` with the Tailwind v4 CLI |
| `make build` | Compile `bin/server` |
| `make up` / `make down` | Start / stop the app + PostgreSQL via docker compose |
| `make migrate` | Apply goose migrations (needs `DATABASE_URL`; used from Story 1.2) |

### Verify the scaffold

```bash
make run
curl -i localhost:8080/healthz   # -> 200 OK, body "ok"
```

Or the full container path (app + PostgreSQL 18):

```bash

```

## Code generation & committed artifacts

`templ` and Tailwind run at **dev time**; their outputs are **committed**:

- `web/**/*_templ.go` — generated by `make templ` (`go tool templ generate`)
- `web/static/css/app.css` — generated by `make css` (Tailwind v4, CSS-first;
  configuration lives in `web/static/css/input.css`, **not** a `tailwind.config.js`)

Because these are committed, the `Dockerfile` only compiles — it carries no
Node or codegen toolchain, producing a single self-contained binary with assets
embedded via `go:embed` (AD-8). Re-run `make generate css` after editing any
`*.templ` file or the Tailwind input before building the image.

## Configuration

Config and secrets come from the **environment only** (AD-8); `internal/config`
loads and validates them at boot, failing fast (and never logging secrets). See
`.env.example`.

| Variable | Required | Used by |
| --- | --- | --- |
| `DATABASE_URL` | yes | pgx pool + goose migrations |
| `SESSION_SECRET` | yes | session signing |
| `OWNER_USERNAME` | yes | the single owner's login name |
| `OWNER_PASSWORD_HASH` | yes | owner's **argon2id PHC hash** (never plaintext) |
| `SECURE_COOKIES` | no (default `false`) | `true` behind HTTPS (Azure); `false` for local http |
| `PORT` | no (default `8080`) | HTTP listen port |

## Database

On startup the server **runs pending goose migrations** (`store.Migrate`,
embedded `db/migrations/*.sql`) **before** serving, then opens a pgx pool
(`store.NewPool`) and pings it — both fail fast. `/healthz` stays
dependency-free; `/readyz` reports database reachability (200/503).

The schema starts with `currency` (seeded USD, BRL — ISO-4217) and a single-row
`app_settings` holding the **Display Currency** (the currency aggregated totals
are shown in; it never alters stored amounts, AD-5). Type-safe queries are
generated by **sqlc** into `internal/store` (`make sqlc`, run via the pinned
`sqlc/sqlc` Docker image). Change the Display Currency at `/settings`.

**Exchange rates** (`/exchange-rates`) are owner-entered, effective-dated, and
**directional** (`from → to`) — stored append-only at `NUMERIC(18,8)`. A lookup
selects the rate effective at (≤) a date; a missing direction is reported (the
owner is prompted) and **never inverted** as `1/rate` (AD-6). `NUMERIC` columns
map to `shopspring/decimal` via the pgx decimal codec + sqlc overrides — the
decimal-money plumbing reused by all future amount/price columns (AD-4).

For local work, bring up just the database (published on host **5433** to avoid
colliding with a native PostgreSQL on 5432):

```bash
docker compose up -d db
export DATABASE_URL=postgres://financas:financas@localhost:5433/financas?sslmode=disable
export SESSION_SECRET=dev-only
make run        # logs: "migrations applied" -> "database connected" -> "listening"
```

Run the DB integration tests by pointing `TEST_DATABASE_URL` (or `DATABASE_URL`)
at that database; they `t.Skip` when neither is set.

## Authentication

Financas serves a **single owner** (AD-7). Every non-login route requires an
authenticated session; the placeholder home page is behind auth.

1. Generate the owner password hash (argon2id) and set it as `OWNER_PASSWORD_HASH`:

   ```bash
   make hashpw            # prompts for a password, prints the hash
   ```

2. Set `OWNER_USERNAME` and `OWNER_PASSWORD_HASH` (plus `SESSION_SECRET`). For
   local docker-compose these default to `owner` / `financas` (dev-only).

3. Sign in at `/login`. Logout is `POST /logout`.

Sessions use a hardened cookie (`HttpOnly`, `SameSite=Lax`, `Secure` when
`SECURE_COOKIES=true`) and expire after **30 min of inactivity** (`IdleTimeout`)
or **12 h absolute** (`Lifetime`). State-changing requests are protected by Go's
built-in `http.CrossOriginProtection` (no per-form token needed). Credentials are
hashed with **argon2id**; nothing sensitive is stored in the cookie.

> Session storage is **in-memory** today (lost on restart; not shared across
> replicas). For Azure Container Apps, run a single replica or upgrade to the
> `scs` postgres store (a one-line store swap + a `sessions` migration).

## App shell & design tokens

After login, every page renders inside a responsive **app shell** (`web/shell.templ`):
a "Welcome back, {owner}" header and top navigation (Dashboard · Investments ·
Transactions · Accounts · Analytics), with a no-JS `<details>` menu on narrow
screens. Nav targets beyond the dashboard are navigable "Coming soon" placeholders
until their epics build them.

The visual system (UX-DR7) lives as **Tailwind v4 `@theme` tokens** in
`web/static/css/input.css`: a semantic palette (`gain` green / `loss` red /
`accent`), neutral `surface`/`ink`/`muted` tones, `rounded-card` (~16px), and a
soft `shadow-card`. Edit the tokens there and rerun `make css`. Story 5.1 extends
this into card/badge/large-number component primitives.

## Money & decimal correctness

All monetary and quantity values use `github.com/shopspring/decimal` — **never
floats** (AD-4, NFR-5; enforced by `make nofloat`). The `internal/money` package
owns the `Money` type (decimal amount + ISO-4217 `Currency`) and `Convert`.
Per AD-12, `Convert` multiplies at full precision and does **not** round;
`Money.Rounded()` applies banker's rounding (half-to-even) to `MoneyScale`
(NUMERIC(19,4)) **once at the display boundary**, so aggregates convert-then-sum
and round only the final total.
